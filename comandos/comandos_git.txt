git status -> mostra o status do diretório git

git add -> adicionar o que quer que vá para o commit (checkpoint)

git commit -m "[mensagem]" -> cria o commit (checkpoin) e cria uma descrição para o mesmo (ex: git commit -m "primeiros comandos bash")

git config --global user.email "[email]" -> vai criar o email do usuário que irá assinar os commits feitos (ex: git config --global user.email "thiagolscardoso@hotmail.com")

git config --global user.name "[name]" -> vai criar o email do usuário que irá assinar os commits feitos (ex: git config --global user.email "Thiago Luz")

git log -> mostra todo o histórico de commit (checkpoint)

## FLUXO DE TRABALHO

1. criar o diretório do projeto
2. iniciar o git dentro do diretório criado (git init .)
3. edita ou cria arquivos
4. git status
5. git add <nome_arquivo>
6. git status
7. git commit -m "mensagem"
8. git status
9. volta para o passo 3.
